DevContainer – Quick Start

Step-by-step guide for Lingaro teams and hackathon participants. In a few minutes, you’ll have a GitHub repo with a ready-to-run DevContainer (GitHub Copilot, Cline, Claude, Qwen) and access to Lingaro GenAI Proxy.

Prerequisites

  • Git + GitHub account, and VS Code with the Dev Containers extension installed.
  • Access to Lingaro GenAI Proxy and a valid API key (will go into .env later).
  • Docker Desktop or Docker Engine running locally.
  • [Windows] git bash installed.

6-Step Setup

  1. Create a GitHub repository (public or private). Recommended name: devcontainer-starter.
    Make sure Issues and Actions are enabled (default settings are fine).
  2. Download the installer get-devcontainer.sh into the root of your repo.
    curl -fsSLo get-devcontainer.sh \
  https://raw.githubusercontent.com/Lingaro/devcontainers-templates/main/hackathon/get-devcontainer.sh
    wget -O get-devcontainer.sh \
  https://raw.githubusercontent.com/Lingaro/devcontainers-templates/main/hackathon/get-devcontainer.sh
    PowerShell (Windows):
    iwr https://raw.githubusercontent.com/Lingaro/devcontainers-templates/main/hackathon/get-devcontainer.sh -OutFile get-devcontainer.sh
  3. Run the installer (grant execute permissions if needed): 
    chmod +x get-devcontainer.sh && ./get-devcontainer.sh
    Note (Windows + Git): If the file opens in Notepad or shows line-ending errors, set UNIX line endings: git config core.autocrlf input and re-download the script.
  4. Fill in environment variables – the installer will create a .devcontainer/ folder with sample files, including .env.example. 
    cp .devcontainer/.env.example .devcontainer/.env
# Open .devcontainer/.env and fill in values
# Most important: API_KEY=<YOUR_LINGARO_GENAI_PROXY_KEY>
    You can obtain your API key using the Lingaro GenAI Proxy Bot on Microsoft Teams: Open Teams Bot.
  5. [Podman only]. If you are using Podman instead of Docker (ie. Docker is not installed):
    1. Podman Desktop. Got to Settings → Preferences → Docker Compatibility. Set Enable for Docker Compatibility.
    2. VS Code. Got to Settings → Extensions → Dev Containers.
      Set Dev > Containers: Docker Compose Path to podman-compose (macOS) or podman compose (Windows).
      Set Dev > Containers: Docker Path to podman.
  6. Open in Dev Container – in VS Code click the Dev Containers icon in the bottom-left corner and select Reopen in Container.
    Or from the Command Palette: ⇧⌘P / Ctrl+Shift+P → “Dev Containers: Reopen in Container”.
  7. Done! After a few moments your DevContainer will start with pre-installed tools: GitHub Copilot, Cline, Claude, Qwen.

Common Issues

  • Docker not running – start Docker Desktop or Docker Engine and retry.
  • Registry access denied – log in if required (e.g. docker login ghcr.io).
  • Permission denied – use chmod +x as in step 3.
  • Missing .env vars – missing GenAI Proxy key will cause LLMs to fail in your workspace.

Launching VS Code from Terminal

If you experience issues launching VS Code from the terminal (e.g., code . command not found), add the appropriate PATH variable:

### For VS Code Stable
# macOS / Linux
export PATH="$PATH:/Applications/Visual Studio Code.app/Contents/Resources/app/bin"

# Windows (PowerShell)
$Env:Path += ";C:\\Users\\%USERNAME%\\AppData\\Local\\Programs\\Microsoft VS Code\\bin"

### For VS Code Insiders
# macOS / Linux
export PATH="$PATH:/Applications/Visual Studio Code - Insiders.app/Contents/Resources/app/bin"

# Windows (PowerShell)
$Env:Path += ";C:\\Users\\%USERNAME%\\AppData\\Local\\Programs\\Microsoft VS Code Insiders\\bin"

How to publish this page with GitHub Pages

  1. Place this file as index.html in your repo (e.g., branch main).
  2. Go to Settings → Pages.
  3. Under Build and deployment choose:
    • Source: Deploy from a branch
    • Branch: main and /(root) (or /docs if files are inside docs/).
  4. Save. After ~1 minute, the site will be live at:
    https://<your-username>.github.io/<repo-name>/

“docs/” Variant

mkdir -p docs && mv index.html docs/
# Settings → Pages → Source: Deploy from a branch → Branch: main /docs

Tip: Secret Files

Never commit your .devcontainer/.env. Add it to .gitignore:

echo ".devcontainer/.env" >> .gitignore

FAQ

Where do I get the Lingaro GenAI Proxy key?

You can obtain it directly via the Lingaro GenAI Proxy Bot in Microsoft Teams. The bot will provide your LLM_API_KEY after authentication.

Can I use an existing repo?

Yes, run steps 2–4 in your existing repo. The .devcontainer/ structure will be added automatically.

“Reopen in Container” option missing in VS Code?

Install the “Dev Containers” extension, restart VS Code, and open the command palette → “Dev Containers: Reopen in Container”.
